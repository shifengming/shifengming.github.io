# v-for

实现迭代数组

<template>
    <div id="app">
        <ol>
            <li v-for="site in sites" :key="site.id">
                {{site.name}}
            </li>
        </ol>
    </div>
</template>
<script>
    export default{
        data(){
            return{
                   sites:[
                       {name: 'Runoob'},
                       {name: 'Google'},
                       {name: 'Taobao'}
                   ]
            }   
        }
    }
</script>

# v-model

v-bind只能实现数据的单向绑定，从M自动绑定到V，无法实现数据的双向绑定

v-model指令则可以实现表单元素和M中数据的双向绑定

<template>
    <div id="app">
        <h4>{{msg}}</h4>
        <input type="text" style="width: 100%;" v-model="msg">
    </div>
</template>
<script>
    export default{
        data(){
            return{
                msg: '星期一'
            }   
        },
    }
</script>

# v-on

事件监听可以使用v-on指令

给标签绑定函数，可缩写为@

<template>
    <div id="app">
         <button @click="greet">Greet</button>
    </div>
</template>
<script>
    export default {
        data() {
            return {
                name: 'Vue.js'
            }   
        },
        methods: {
            greet() {
                // `this` 在方法里指当前 Vue 实例
                alert(`My age is  ${this.name}  and I love  ${this.name}  !`)
            }
        }
    }
</script>

# v-once

<template>
    <div id="app">
         <input type="text" v-model="content">
         <p v-once>输入内容:{{content}}</p>
    </div>
</template>
<script>
    export default {
        data() {
            return {
                content: "早上好啊"
            }   
        }
    }
</script>

只渲染一次，后面元素中的数据再更新变化，都不会重新渲染，例如下面：给

标签添加了v-once指令，并且content赋了初始值，第一次渲染的时候input控件和p标签都有了初始值，后面我们再更新content值的时候，p标签的内容不会发生变化

# v-else

和v-if指令搭配使用，没有对应的值，当v-if的值false，v-else才会被渲染出来

<template>
    <div id="app">
        <p v-if="show1">show</p>
        <p v-else>v-else</p>
    </div>
</template>
<script>
    export default {
        data() {
            return {
                show1: false,
            }   
        }
    }
</script>

# v-cloak

这个一般是用在获取后台数据后渲染到页面上。有可能由于网络慢的原因数据还没获取完就加载了页面导致页面上的值为undefined

v-cloak指令是解决屏幕闪动的好方法

<template>
    <div id="app" v-cloak>
          {{context}}
    </div>
</template>
<script>
    export default {
        data() {
            return {
                 context:'互联网头部玩家钟爱的健身项目'
            }   
        }
    }
</script>
<style>
[v-cloak]{
    display: none;
}
</style>

互联网头部玩家钟爱的健身项目
